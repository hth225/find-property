# 전세/월세 사기 점검 모듈

> 본 점검은 참고용이며 법률·중개 자문이 아닙니다. 계약 전 반드시 등기부등본·건축물대장 확인, 공인중개사·HUG·법률 전문가 상담을 거치세요.

이 모듈은 find-property 스킬이 전세/월세 매물을 채점하기 전에 로드되며, Tier-2 체크리스트는 DUE-DILIGENCE 단계 핸드오프 시 제시됩니다. 주요 피해층은 20-30대이며, 보증금 손실은 회복이 어렵습니다. 데이터로 확인 불가한 항목은 추측하지 말고 명시적으로 "확인 불가"로 표기하고 Tier-2 실사로 넘깁니다.

## Tier 점검 체계

### Tier-1 — 관찰 가능한 매물 데이터 기반 예비 플래그

매물 리스팅 단계에서 관찰 가능한 데이터만으로 산정하는 예비 신호입니다. 단정이 아니라 advisory 플래그이며, 모든 항목은 Tier-2 서류 실사로 검증되어야 합니다.

- 🚩 **전세가율 위험**: (보증금 + 선순위 채권 추정) / 매매시세 가 높음 (깡통전세). 매매시세가 빌라/오피스텔은 자주 불명확 → 산정 불가 시 "전세가율 산정 불가 — 시세 불투명 자체가 위험신호"로 처리 (값을 추측하지 말 것).
- 🚩 **시세 대비 이상**: 가격이 MOLIT 지역 실거래가 comps 대비 과도 (area-level only; advisory).
- 🔴 **근린생활시설(근생빌라)**: 주거용 위반 → 이행강제금·원상복구 리스크, HUG 보증·전세대출 제약. ("전입신고 불가"로 단정하지 말 것 — 가능할 수 있음.)
- 🔴 **동시진행 신축빌라 패턴**: 신축 + 전세가≈매매가 + 분양·중개 일괄 + 소유권 이전 임박 → 무자본 갭투자/바지임대인 위험. (20-30대 주요 피해 벡터 — minor 아닌 🔴로 취급.)
- ⚠️ **다주택/비아파트 임대인 패턴** (탐지 가능 시).

### Tier-1.5 — 매물 데이터로 확인 불가한 고위험 벡터 (Tier-2 강제)

다음 벡터들은 매물 리스팅 데이터만으로는 탐지가 불가능하지만, 실제 보증금 손실의 핵심 원인입니다. 해당 매물은 무조건 Tier-2 실사로 강제 핸드오프합니다.

- **신탁사기**: 신탁사 명의 → 임대인에게 임대권한 없음, 전입+확정일자에도 대항력 無. 등기부 신탁 표시 시 단독으로 🔴 + 신탁사 동의 확인 필수.
- **이중·대리계약**: 대리인 계약 시 위임장+인감증명 확인, 보증금은 등기부상 소유자 본인 계좌로만 입금.
- **다가구 선순위 보증금 합산**: 다가구주택은 전체 세대 보증금 + 근저당 합산이 시세 초과 시 후순위 전액 손실 가능.
- **익일 0시 대항력 공백**: 전입신고+확정일자는 다음날 0시부터 효력 → 입주일 당일 근저당 설정으로 추월당함. 대응: 특약(입주 익일까지 근저당 설정 금지), 전세권설정등기, 또는 잔금일 등기 확인.

### Tier-2 — 계약 전 서류 실사 체크리스트

계약 전 반드시 직접 확인해야 할 서류와 사실 관계입니다.

- **등기부등본**: 근저당 규모·압류·가압류·임의경매·신탁 여부·주소·면적 일치 확인.
- **건축물대장**: 위반건축물 여부·용도(주거/근생) 확인.
- **임대인 세금**: 국세·지방세 미납 여부 확인.
- **공인중개사**: 자격 + 중개사무소 정상영업 + 공제(보증)가입 확인.
- **선순위 보증금**: 전세권/확정일자 선순위 보증금 + (다가구) 전 세대 보증금 합산 확인.
- **대리계약**: 위임장·인감증명·소유자 본인계좌 입금 확인.
- **HUG 전세보증금 반환보증**: 가입 가능 여부 확인 (대략 공시가격×140% / 전세가율 ~90% 기준 — 정확 기준은 build 시 HUG 현행 기준 확인).
- **안심전세앱**: 빌라·오피스텔 시세·임대인 보증사고·체납·악성임대인 이력 조회.
- **계약서**: 주택임대차표준계약서 + 특약.
- **대항력 확보**: 전입신고 + 확정일자.

### 월세 주의사항

- **임대인 확인**: 대리인 회피, 소유자 본인계좌 입금.
- **선순위 부담**: 근저당 + 보증금 ≤ 매매가 70~80%.
- 🚩 **관리비 투명성**: 실질 임대료 은폐 신호.
- **옵션 상태** 확인.
- **위반건축물** 확인.
- **임차인 보호 특약** 확보.
- **중개수수료**: 환산보증금 요율 확인.
- **잔금·확정일자·전입신고** + 익일 0시 주의.

### 매매 due-diligence

- **근저당 말소 조건 특약** 명시.
- **잔금·소유권이전 동시이행**.
- **가등기·가압류 확인**.
- **실거래가 대비 가격** 검토.
- **재건축/재개발 권리관계** 확인.

### Output contract

출력 객체: `{ badge, triggered_flags[], unverifiable_high_severity[], recommended_actions[], disclaimer }`

Badge 라벨 (절대 "확정 무사고" 류 표현을 쓰지 말 것 — 본 점검은 예비 점검일 뿐):

- 🟢 **예비점검 특이사항 없음 (등기부 확인 필수)** — 위험신호 없음(서류 확인 전). 예비 데이터에서 플래그가 없을 뿐이며, Tier-2 실사로만 확정.
- 🟡 **주의 — 확인 필요** — 하나 이상의 🚩/⚠️ 플래그 존재, Tier-2 정밀확인 필요.
- 🔴 **위험 — 계약 전 정밀확인** — 다음 trigger 발생 시.
  - 🔴 triggers: 깡통전세 / 근생빌라 / 동시진행 신축빌라 / 신탁 / severe flag.

`disclaimer` 필드에는 항상 상단 disclaimer 전문을 포함합니다. `unverifiable_high_severity[]`에는 Tier-1.5 벡터 중 데이터로 확인 불가한 항목을 담아 Tier-2 실사로 강제합니다.
